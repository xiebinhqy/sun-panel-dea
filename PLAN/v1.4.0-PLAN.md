## 一、功能规划总览

### 1. OpenAPI 开放接口（beta）
- **状态**: ⬜ 待开发
- **优先级**: 🔴 P0
- **描述**: 提供 RESTful API 接口，开发者可通过 API 操作数据
- **参考**: 标准的 OpenAPI/Swagger 文档
- **实现方案**:
  - 后端: Go + gin 路由 + swagger 文档
  - 前端: 新增 OpenAPI 管理页面（在系统应用中）
  - 认证: 基于 Token 的 API 鉴权

### 2. 全局站点设置
- **状态**: ⬜ 待开发
- **优先级**: 🔴 P0
- **包含功能**:
  - [ ] 自定义站点标题 PRO
  - [ ] 自定义站点图标 PRO
  - [ ] 自定义登录页面背景图
- **实现方案**:
  - 在后端 panelConfig 结构体中新增字段
  - 前端 Settings 页面新增对应的配置项

### 3. 在线编辑 index.js / index.css
- **状态**: ⬜ 待开发
- **优先级**: 🟡 P1
- **描述**: 
  - PRO 用户：可在 Web 界面在线编辑
  - 非 PRO 用户：依旧从程序安装目录修改
- **实现方案**:
  - 新建 API 路由: GET/PUT `/api/custom/file`
  - 存储路径: `conf/custom/index.js` 和 `conf/custom/index.css`
  - 前端新建在线编辑器页面（使用 monaco-editor 或 codemirror）

### 4. 背景图格式支持扩展
- **状态**: ⬜ 待开发
- **优先级**: 🟡 P1
- **描述**: 支持 heic、avif 格式上传
- **实现方案**: 修改后端文件上传校验逻辑

### 5. 功能修复
- **状态**: ⬜ 待开发
- **优先级**: 🔴 P0
- **修复项**:
  - [ ] v1.4.0-beta 旧版升级不兼容在线编辑 JS/CSS 的问题

### 6. 功能优化
- **状态**: ⬜ 待开发
- **优先级**: 🟢 P2
- **优化项**:
  - [ ] 禁用 referrer（解决 qBittorrent 等跳转问题）
  - [ ] 优化获取三方网站图标
  - [ ] 编辑项目时网址检测（未以 http/https 开头提醒）
  - [ ] 修改系统状态详情图标显示格式（硬盘/内存: 已使用量/总量）

### 7. Docker 精简挂载目录
- **状态**: ⬜ 待开发
- **优先级**: 🟡 P1
- **描述**: 精简 docker 挂载目录为单个 `./conf`
- **兼容性**: 仍然兼容旧版本的多路径挂载方式

## 二、技术方案

### 后端架构
```
service/api/api_v1/
├── system/
│   ├── update.go        # 在线更新（已有）
│   ├── openapi.go       # [新增] OpenAPI 接口
│   └── custom.go        # [新增] 自定义 JS/CSS 编辑
├── panel/
│   └── config.go        # [修改] 站点设置接口
```

### 前端组件
```
src/components/apps/
├── OpenApi/             # [新增] OpenAPI 管理页面
│   └── index.vue
├── CustomEditor/        # [新增] 在线编辑器
│   └── index.vue
├── Style/               # [修改] 站点设置新增选项
│   └── index.vue
```

### 数据库/配置
```
conf/
├── custom/
│   ├── index.js         # 自定义 JS
│   └── index.css        # 自定义 CSS
```
